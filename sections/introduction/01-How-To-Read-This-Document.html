This document is a detailed specification for FAIR Digital Objects (FDOs). The document is primarily intended for the following audiences:

  • Anyone who wants to understand the basic ideas of the FDO concept.
  • Software developers who want to implement the FDO concept using a specific technology.
  • Software developers who want to implement services that support the creation, manipulation and management of FDOs.
  • Software developers who want to implement clients that are able to process and interact with FDOs.
This specification document is structured in the following way:
  • Section 1 is the introduction and gives a high-level overview of the FDO Concept.
  • Section 2 includes a glossary of terms which are being used in this document and in many FDO related discussions.
  • Section 3 is the core of this document, since it specifies in detail the components of the architecture and their characteristics. This chapter is normative.
  • Section 4 details the steps a client would take to process an FDO.
  • Section 5 describes conformance criteria.
  • Chapter 6 includes corrections that have been done after each version upgrade.
